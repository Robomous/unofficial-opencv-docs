Para una computadora, una imagen no es una fotografía o una pintura—es simplemente una cuadrícula de números.

Entender esta estructura numérica es la clave de la visión por computadora. Una vez que dejas de ver "fotos" y empiezas a ver "matrices", operaciones como la detección de bordes, filtrado de color y reconocimiento de objetos se convierten simplemente en problemas matemáticos.

## La Cuadrícula de Píxeles

Imagina hacer zoom en una foto hasta que se convierte en un mosaico de bloques. Cada uno de esos bloques es un **píxel** (Elemento de Imagen), el átomo fundamental de una imagen digital.

- **Coordenadas**: En OpenCV, referenciamos los píxeles por `(y, x)` o `(fila, columna)`. Esto es un poco contraintuitivo si estás acostumbrado a las coordenadas cartesianas `(x, y)`, pero es el estándar para matrices.
- **Resolución**: Simplemente las dimensiones de la cuadrícula. Una imagen de 1920x1080 es una cuadrícula de 1080 filas de alto por 1920 columnas de ancho.

## Bajo el Capó: Es Solo una Matriz

Cuando cargas una imagen en OpenCV, no obtienes un "Objeto Imagen" especial—obtienes un **arreglo de NumPy** (NumPy array).

Esta es una de las decisiones de diseño más poderosas de OpenCV. Al usar arreglos estándar de NumPy, las imágenes de OpenCV son compatibles con todo el ecosistema científico de Python (como Matplotlib, Scikit-learn y Pandas) desde el primer momento. No necesitas convertidores especiales; si sabes cómo trabajar con listas de datos en Python, ya sabes cómo editar imágenes.

<Aside type="tip">
  ¿Nuevo en NumPy? Es la biblioteca estándar para computación numérica en
  Python. Echa un vistazo al [NumPy Quickstart
  oficial](https://numpy.org/doc/stable/user/quickstart.html) para aprender lo
  básico.
</Aside>

### Escala de Grises: Una Hoja 2D

Una imagen en blanco y negro (escala de grises) es la forma más simple. Es una **matriz 2D** (piensa en una hoja de Excel) donde cada celda representa la intensidad de un solo píxel.

- **0** = Negro Puro (Sin luz)
- **255** = Blanco Puro (Máxima luz)
- **128** = Gris Medio

Los algoritmos de visión por computadora a menudo convierten imágenes a color a escala de grises primero. ¿Por qué? Porque para tareas como detectar bordes o leer texto, solo te interesan las formas de intensidad de luz, no el color. Eliminar el color reduce los datos en un 66%, haciendo que tu código se ejecute significativamente más rápido.

```text
Matriz de Imagen (8-bit escala de grises), 3x3 píxeles:
┌─────────────────┐
│   0   128   255 │  <- Fila 0 (Negro -> Gris -> Blanco)
│ 255   128     0 │  <- Fila 1 (Blanco -> Gris -> Negro)
│  50    50    50 │  <- Fila 2 (Línea Gris Oscuro)
└─────────────────┘
```

### Color: Una Pila 3D

Las imágenes a color añaden una tercera dimensión. En lugar de una sola hoja de números, imagina tres hojas apiladas una encima de la otra. Cada hoja representa un **canal de color**: típicamente **Azul**, **Verde** y **Rojo**.

Entonces, un solo píxel no es solo un número—es un **vector de tres números** (B, G, R).

- **Píxel Azul Puro**: `[255, 0, 0]` (Máx Azul, Sin Verde, Sin Rojo)
- **Píxel Rojo Puro**: `[0, 0, 255]` (Sin Azul, Sin Verde, Máx Rojo)
- **Píxel Blanco**: `[255, 255, 255]` (Máx de todos los colores)
- **Píxel Negro**: `[0, 0, 0]` (Sin luz en absoluto)

Esta estructura `(Alto, Ancho, Canales)` es la razón por la que verás formas como `(1080, 1920, 3)`.

## Código: Viendo la Matriz

Así es como se ve esto en código Python real. Nota que "cargar una imagen" es realmente solo "cargar una matriz desde un archivo".

```python
import cv2
import numpy as np

# Cargar una imagen
img = cv2.imread('foto.jpg')

# 1. VERIFICAR LA FORMA (SHAPE)
# .shape da (Filas/Alto, Columnas/Ancho, Canales)
# Si retorna (500, 500), es escala de grises (sin dimensión de canales).
# Si retorna (500, 500, 3), es una imagen a color.
print(f"Forma: {img.shape}")

# 2. LEER UN PÍXEL
# Miremos el píxel en Fila 50, Columna 100
px = img[50, 100]
print(f"Valor del píxel en (50, 100): {px}")
# Salida: [240  50  50]  -> Aproximadamente equivalente a Azul=240, Verde=50, Rojo=50

# 3. MODIFICAR UN PÍXEL
# Cambiar el píxel a Azul puro
img[50, 100] = [255, 0, 0] # [Azul, Verde, Rojo]
```

### Creando Imágenes desde Cero

Dado que las imágenes son solo arreglos, no necesitamos cargar un archivo para tener una imagen. Podemos crear una matemáticamente usando `np.zeros()` (que crea una matriz llena de 0s).

¿Por qué 0s? Porque 0 es negro. Así que `np.zeros()` efectivamente crea un lienzo negro.

```python
# 1. Crear un lienzo negro
# Dimensiones: 480px alto, 640px ancho, 3 canales (Color)
# dtype=np.uint8: ¡CRÍTICO! Las imágenes esperan enteros de 8 bits (0-255).
# Si usas flotantes, OpenCV podría confundirse.
lienzo = np.zeros((480, 640, 3), dtype=np.uint8)

# 2. Dibujar una línea Azul a través del medio
# Establecer Fila 240, Todas las Columnas (:), Canal 0 (Azul) a 255
lienzo[240, :, 0] = 255
```

## Profundidad de Color (Bit Depth)

Habrás notado que seguimos mencionando "0 a 255". ¿Por qué ese rango específico?

Se reduce a la eficiencia de la memoria. Una imagen estándar usa **8 bits** (1 byte) para almacenar cada valor de color.

$$
2^8 = 256 \text{ posibilidades}
$$

- **8-bit (uint8)**: El estándar. 0-255 por canal. Esto nos da 16.7 millones de colores ($256 \times 256 \times 256$), que es más de lo que el ojo humano puede distinguir.
- **16-bit / 32-bit (flotante)**: A veces 256 pasos no son suficientes. Por ejemplo, en **Imágenes Médicas (Rayos X)** o **Fotografía HDR**, podrías necesitar distinguir entre "gris oscuro" y "gris oscuro ligeramente más claro" con extrema precisión. En esos casos, usamos enteros de 16 bits ($0-65,535$) o números de punto flotante de 32 bits ($0.0-1.0$).

<Aside type="note">
  **Raster vs Vector**: OpenCV trabaja con imágenes **Raster** (cuadrículas de
  píxeles). Los formatos vectoriales (como SVG o AI) son ecuaciones matemáticas,
  no cuadrículas, y deben ser rasterizados (convertidos a píxeles) antes de que
  OpenCV pueda tocarlos.
</Aside>

## Por Qué Esto Importa

Entender esta estructura explica por qué:

1.  **OpenCV es rápido**: Usa las operaciones de arreglos de NumPy altamente optimizadas por debajo.
2.  **Las coordenadas difieren**: Las matrices matemáticas usan `(Fila, Columna)` vs Cartesiano `(x, y)`.
3.  **Existen espacios de color**: Cambiar de BGR a Escala de Grises es simplemente combinar matemáticamente los 3 canales en 1.

Ahora que conoces la matriz, estás listo para empezar a manipularla.
