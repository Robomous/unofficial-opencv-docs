## Tu Primer Script de OpenCV

Comencemos con el "Hola Mundo" de la Visión por Computadora: cargar una imagen, mostrarla y guardar una copia.

```python
import cv2

# 1. Leer una imagen desde disco
# Retorna un arreglo de NumPy, o None si el archivo no existe
img = cv2.imread("image.jpg")

if img is None:
    raise FileNotFoundError("No se pudo cargar la imagen. ¡Verifica tu ruta!")

# 2. Mostrar la imagen en una ventana
# "Imagen original" es el título de la ventana
cv2.imshow("Imagen original", img)

# 3. Esperar indefinidamente una tecla
# 0 significa "esperar por siempre". Si pasas 1000, espera 1 segundo.
cv2.waitKey(0)

# 4. Limpiar
cv2.destroyAllWindows()

# 5. Guardar el resultado
cv2.imwrite("image_copy.jpg", img)
```

Este script simple introduce la "Santísima Trinidad" del I/O en OpenCV: `imread`, `imshow` e `imwrite`. Las usarás en casi todos los proyectos.

## Conceptos Clave (Las "Trampas")

Antes de avanzar, hay tres peculiaridades que necesitas internalizar para evitar horas de depuración más tarde.

<Steps>

1.  **OpenCV es BGR, no RGB**

    Este es un artefacto histórico. Cuando se creó OpenCV, **BGR** (Azul-Verde-Rojo) era un diseño de memoria popular para cámaras. Hoy, casi todos (navegadores, matplotlib, tu teléfono) usan **RGB**.

    Si tus imágenes se ven "pitufas" (personas azules, cielos naranjas), probablemente olvidaste convertir:

    ```python
    img_bgr = cv2.imread("image.jpg")
    img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB) # Arregla colores para Matplotlib/Web
    ```

2.  **Las Imágenes son solo Arreglos de NumPy**

    No hay una "Clase Imagen" especial en OpenCV Python. Solo existe `numpy.ndarray`. Esta es una noticia fantástica porque significa que puedes usar todas las herramientas del ecosistema NumPy en tus imágenes.

    ```python
    import cv2
    import numpy as np

    img = cv2.imread("image.jpg")

    print(type(img))       # <class 'numpy.ndarray'>
    print(img.shape)       # (alto, ancho, canales) -> ej. (1080, 1920, 3)
    print(img.dtype)       # uint8 (Entero sin signo de 8 bits)
    ```

    <Aside type="tip">
      **Modelo Mental**: Piensa en una imagen como una matriz 3D genérica de
      números. - shape: `(Filas/Alto, Columnas/Ancho, Canales/Colores)` - dtype:
      `uint8` (0-255). 0 es negro, 255 es brillante.
    </Aside>

3.  **(Y, X) no (X, Y)**

    Debido a que son matrices, accedemos a ellas por `[Fila, Columna]`. En términos de imagen, la `Fila` es el **eje Y** (alto) y la `Columna` es el **eje X** (ancho).

    ```python
    # Accediendo al píxel en y=10, x=20
    pixel = img[10, 20]
    ```

    Si intentas `img[x, y]`, te estrellarás u obtendrás el píxel incorrecto. Siempre piensa **"Fila, Columna"**.

</Steps>

## Operaciones Básicas de Imagen

Dado que las imágenes son solo arreglos, "editar" una imagen es realmente solo "rebanar" (slicing) un arreglo.

### 1. Conversión a Escala de Grises

La reducción de información es clave. El color a menudo añade ruido sin añadir valor (ej: para detectar bordes).

```python
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)
```

### 2. Redimensionamiento (Resize)

Las imágenes más pequeñas se procesan más rápido. A menudo, una imagen 4K es excesiva para detección; redimensionar a 640x480 es estándar.

```python
# Tamaño fijo (Ancho, Alto) <- Nota: ¡resize usa el orden (Ancho, Alto)!
resized = cv2.resize(img, (640, 480))

# Escalado relativo
half_size = cv2.resize(img, None, fx=0.5, fy=0.5)
```

### 3. Recorte (Cropping)

No se necesita una función especial. ¡Solo rebana el arreglo de NumPy!

```python
# Recortar [FilaInicio:FilaFin, ColInicio:ColFin]
# AKA [y1:y2, x1:x2]
crop = img[100:300, 200:500]
```

### 4. Dibujo

Anota tus resultados. Útil para depurar o mostrar al usuario lo que la IA vio.

```python
# Dibujar un Rectángulo Verde (BGR: 0, 255, 0)
cv2.rectangle(img, (50, 100), (250, 250), (0, 255, 0), 2)

# Escribir Texto
cv2.putText(img, "Objetivo Detectado", (50, 420), cv2.FONT_HERSHEY_SIMPLEX, 1.0, (255, 255, 255), 2)
```

## Filtros y Características

Ahora hagamos algo de visión por computadora real.

### Desenfoque (Blurring)

¿Por qué desenfocar? Las cámaras producen "ruido" (grano). El desenfoque suaviza ese ruido para que los algoritmos no confundan el grano con bordes.

```python
# El tamaño del kernel (15, 15) determina qué tan fuerte es el desenfoque
blur = cv2.GaussianBlur(img, (15, 15), 0)
```

### Detección de Bordes (Canny)

El algoritmo **Canny** encuentra dónde la intensidad del píxel cambia rápidamente (gradientes). Es básicamente un generador de dibujos lineales.

```python
# Los umbrales inferior/superior determinan qué tan exigente es el algoritmo
edges = cv2.Canny(gray, 100, 200)
```

## Trabajando con Video

El video es solo una secuencia de imágenes (frames) mostrada rápido. Si puedes procesar una imagen, puedes procesar video.

### Loop de Captura de Webcam

```python
cap = cv2.VideoCapture(0) # 0 es usualmente tu webcam por defecto

if not cap.isOpened():
    raise RuntimeError("No se pudo abrir la webcam.")

while True:
    # 1. Tomar un frame
    ret, frame = cap.read()
    if not ret: break # Detener si el stream termina

    # 2. Procesar el frame (ej: convertir a gris)
    processed = cv2.cvtColor(frame, cv2.COLOR_BGR2GRAY)

    # 3. Mostrarlo
    cv2.imshow("Webcam Feed", processed)

    # 4. Salir si se presiona 'q'
    if cv2.waitKey(1) & 0xFF == ord('q'):
        break

cap.release()
cv2.destroyAllWindows()
```

## Poniéndolo Todo Junto: Un Mini-Pipeline

Combinemos todo en un script útil: detectar formas.

1.  **Entrada**: Imagen
2.  **Pre-proceso**: Escala de Grises -> Desenfoque (eliminar ruido)
3.  **Proceso**: Bordes Canny -> Encontrar Contornos
4.  **Post-proceso**: Dibujar cajas
5.  **Salida**: Imagen con detecciones

```python
import cv2

# 1. Cargar
img = cv2.imread("shapes.png")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# 2. Desenfocar para eliminar ruido
blur = cv2.GaussianBlur(gray, (5, 5), 0)

# 3. Detectar Bordes
edges = cv2.Canny(blur, 50, 150)

# 4. Encontrar Contornos (curvas continuas de puntos)
contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

# 5. Dibujar resultados
output = img.copy()
for cnt in contours:
    # Obtener la caja delimitadora del contorno
    x, y, w, h = cv2.boundingRect(cnt)
    # Dibujarla
    cv2.rectangle(output, (x, y), (x + w, y + h), (0, 255, 0), 2)

cv2.imshow("Detecciones", output)
cv2.waitKey(0)
cv2.destroyAllWindows()
```

## Detección de Rostros (Haar Cascades)

Para una victoria rápida, OpenCV viene con modelos pre-entrenados. **Haar Cascades** son una técnica más antigua (de 2001), pero son rápidos y corren bien en CPUs.

```python
# Cargar el XML del modelo pre-entrenado
face_cascade = cv2.CascadeClassifier(
    cv2.data.haarcascades + "haarcascade_frontalface_default.xml"
)

img = cv2.imread("people.jpg")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# La Línea Mágica
faces = face_cascade.detectMultiScale(gray, scaleFactor=1.1, minNeighbors=4)

# Dibujar rostros
for (x, y, w, h) in faces:
    cv2.rectangle(img, (x, y), (x + w, y + h), (255, 0, 0), 2)
```

<Aside type="note">
  Para reconocimiento facial en producción hoy en día, probablemente usarías
  Deep Learning (como el módulo `dnn` en OpenCV o bibliotecas como `dlib`), pero
  Haar Cascades son excelentes para aprender los conceptos.
</Aside>

## Errores Comunes

Incluso los profesionales cometen estos errores. Revisa esta lista si tu código actúa extraño:

1.  **Problemas de Ruta**: `cv2.imread()` no da error si falta el archivo; simplemente retorna `None`. Siempre verifica `if img is None:`.
2.  **Confusión de color**: Recuerda **BGR**. Si tus colores se ven intercambiados, probablemente lo están.
3.  **Tipos de Datos**: Las imágenes son `uint8` (0-255). Si haces matemáticas y terminas con flotantes (0.0-1.0), `imshow` podría mostrar un cuadrado negro a menos que conviertas de vuelta o normalices.
4.  **Loops Infinitos**: Si tu ventana de video se congela, probablemente olvidaste `cv2.waitKey(1)`. Esta función es la que realmente le dice al SO que dibuje los píxeles de la ventana.
