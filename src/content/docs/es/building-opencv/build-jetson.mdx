## Referencia de Hardware

Especificaciones del **Jetson Nano Developer Kit** relevantes para esta compilación:

| Especificación | Valor |
|---|---|
| SoC | NVIDIA Tegra X1 (tegra210) |
| GPU | Arquitectura Maxwell, 128 núcleos CUDA |
| Compute capability CUDA | **5.3** |
| RAM | 4 GB LPDDR4 (compartida entre CPU y GPU) |
| Último JetPack soportado | 4.6.x |

**Stack de software de JetPack 4.6.x:**

| Componente | Versión |
|---|---|
| SO | Ubuntu 18.04 (L4T 32.7.x) |
| CUDA | **10.2** |
| cuDNN | **8.2** |
| TensorRT | 8.2 |
| OpenCV preinstalado | 4.1.1 — **sin soporte CUDA** |

<Aside type="danger">
El OpenCV preinstalado del Jetson Nano (4.1.1 de JetPack) **no tiene soporte CUDA**. Debes compilar desde el código fuente para obtener OpenCV con aceleración CUDA. `pip install opencv-python` y `sudo apt install libopencv-dev` también proporcionan versiones sin CUDA e interferirán con una instalación compilada desde fuente.
</Aside>

## Compilación Paso a Paso

<Steps>
1. **Ampliar la memoria swap.**

   El Jetson Nano tiene 4 GB de RAM compartida entre CPU y GPU. Compilar OpenCV 4.6+ requiere al menos 8.5 GB de memoria total (RAM + swap) para una compilación paralela de 4 núcleos. Con solo los 2 GB predeterminados de zram swap, la compilación fallará con errores OOM o recurrirá a compilación de un solo núcleo.

   ```bash
   # Verificar memoria actual — necesitas RAM + Swap > 8500 MB para una compilación rápida de 4 núcleos
   free -m

   # Deshabilitar el swap zram existente
   sudo swapoff -a

   # Crear un archivo swap de 6 GB en la tarjeta microSD (o unidad USB — más rápido)
   sudo fallocate -l 6G /var/swapfile
   sudo chmod 600 /var/swapfile
   sudo mkswap /var/swapfile
   sudo swapon /var/swapfile

   # Hacer permanente (añadir a fstab)
   echo '/var/swapfile swap swap defaults 0 0' | sudo tee -a /etc/fstab

   # Verificar
   free -m
   ```

2. **Configurar el Jetson en modo de máximo rendimiento.**

   ```bash
   sudo jetson_clocks       # fijar relojes al máximo
   sudo nvpmodel -m 0       # modo MAXN (los 4 núcleos CPU, GPU completa)
   ```

3. **Eliminar el OpenCV preinstalado** para evitar conflictos de importación.

   ```bash
   sudo apt-get purge libopencv-dev libopencv python3-opencv
   sudo apt-get autoremove
   sudo apt-get update
   ```

4. **Verificar que las variables de entorno de CUDA estén configuradas.**

   CUDA ya debería estar en el PATH gracias a JetPack, pero confirma:

   ```bash
   nvcc --version   # debe mostrar CUDA 10.2
   python3 -c "import ctypes; ctypes.cdll.LoadLibrary('libcuda.so.1')"
   ```

5. **Instalar dependencias de compilación.**

   ```bash
   sudo apt-get update && sudo apt-get upgrade -y

   sudo apt-get install -y \
     build-essential cmake git pkg-config \
     libavcodec-dev libavformat-dev libswscale-dev \
     libgstreamer1.0-dev libgstreamer-plugins-base1.0-dev \
     libxvidcore-dev libx264-dev \
     libgtk-3-dev libcanberra-gtk3-module libcanberra-gtk-module \
     libjpeg-dev libpng-dev libtiff-dev \
     libtbb2 libtbb-dev \
     libdc1394-22-dev \
     libv4l-dev v4l-utils \
     libhdf5-dev libhdf5-serial-dev \
     libopenblas-dev liblapack-dev libblas-dev gfortran \
     python3-dev python3-numpy python3-pip \
     libopenjp2-7-dev

   # Cabeceras de desarrollo de cuDNN (ya instaladas por JetPack 4.6)
   sudo apt-get install -y libcudnn8-dev
   ```

6. **Clonar los repositorios** — fija los tres al mismo tag de versión.

   ```bash
   OPENCV_VERSION="4.10.0"

   cd ~
   git clone --branch ${OPENCV_VERSION} https://github.com/opencv/opencv.git
   git clone --branch ${OPENCV_VERSION} https://github.com/opencv/opencv_contrib.git
   git clone https://github.com/opencv/opencv_extra.git
   cd opencv_extra && git checkout ${OPENCV_VERSION} && cd ..

   mkdir ~/opencv/build && cd ~/opencv/build
   ```

7. **Configurar con CMake** (JetPack 4.6, CUDA 10.2).

   ```bash
   cmake \
     -D CMAKE_BUILD_TYPE=RELEASE \
     -D CMAKE_INSTALL_PREFIX=/usr \
     -D OPENCV_EXTRA_MODULES_PATH=~/opencv_contrib/modules \
     -D OPENCV_TEST_DATA_PATH=~/opencv_extra/testdata \
     -D WITH_CUDA=ON \
     -D WITH_CUDNN=ON \
     -D OPENCV_DNN_CUDA=ON \
     -D CUDA_ARCH_BIN=5.3 \
     -D CUDA_ARCH_PTX="" \
     -D ENABLE_FAST_MATH=ON \
     -D CUDA_FAST_MATH=ON \
     -D WITH_CUBLAS=ON \
     -D ENABLE_NEON=ON \
     -D WITH_GSTREAMER=ON \
     -D WITH_LIBV4L=ON \
     -D WITH_TBB=ON \
     -D WITH_GTK=ON \
     -D WITH_OPENGL=ON \
     -D WITH_QT=OFF \
     -D BUILD_opencv_python3=ON \
     -D PYTHON3_EXECUTABLE=$(which python3) \
     -D PYTHON3_NUMPY_INCLUDE_DIRS=$(python3 -c "import numpy; print(numpy.get_include())") \
     -D PYTHON3_PACKAGES_PATH=/usr/lib/python3/dist-packages \
     -D BUILD_TESTS=OFF \
     -D BUILD_PERF_TESTS=OFF \
     -D BUILD_EXAMPLES=OFF \
     -D OPENCV_GENERATE_PKGCONFIG=ON \
     ..
   ```

   **Notas sobre flags clave:**
   - `ENABLE_NEON=ON` — La CPU ARM Cortex-A57 soporta instrucciones SIMD NEON; este flag proporciona mejoras de rendimiento significativas en el lado CPU
   - `CMAKE_INSTALL_PREFIX=/usr` — Mantiene la instalación coherente con otros componentes de JetPack
   - `PYTHON3_PACKAGES_PATH=/usr/lib/python3/dist-packages` — Requerido en Jetson; sin él, el archivo `.so` de Python se instala donde Python no puede encontrarlo automáticamente
   - `BUILD_TESTS=OFF`, `BUILD_EXAMPLES=OFF` — Desactivados para ahorrar tiempo de compilación y espacio en disco en el Nano

8. **Compilar e instalar.**

   ```bash
   # Compilar usando los 4 núcleos (requiere el swap ampliado del Paso 1)
   make -j4

   # Si se queda sin memoria, recurre a un solo núcleo:
   # make -j1

   # Instalar
   sudo make install
   sudo ldconfig

   # Verificar que Python encuentra el nuevo cv2
   python3 -c "import cv2; print(cv2.__version__); print(cv2.getBuildInformation())"
   ```

   Busca `NVIDIA CUDA: YES` y `cuDNN: YES` en la salida de `getBuildInformation()`.

9. **Verificar que CUDA funciona.**

   ```python
   import cv2

   # Verificar el número de dispositivos CUDA — debe ser 1 en el Jetson Nano
   count = cv2.cuda.getCudaEnabledDeviceCount()
   print(f"CUDA devices: {count}")

   # Imprimir solo las líneas relacionadas con CUDA de la información de compilación
   info = cv2.getBuildInformation()
   for line in info.split('\n'):
       if 'CUDA' in line or 'cuDNN' in line:
           print(line)

   # Prueba con pipeline GStreamer (específico de Jetson, cámara CSI)
   pipeline = (
       "nvarguscamerasrc ! "
       "video/x-raw(memory:NVMM), width=1280, height=720, framerate=60/1, format=NV12 ! "
       "nvvidconv flip-method=0 ! "
       "video/x-raw, width=1280, height=720, format=BGRx ! "
       "videoconvert ! "
       "video/x-raw, format=BGR ! "
       "appsink"
   )
   # cap = cv2.VideoCapture(pipeline, cv2.CAP_GSTREAMER)
   # print(f"CSI camera opened: {cap.isOpened()}")
   ```

10. **Limpieza post-instalación** (importante para la longevidad de la microSD).

    ```bash
    # Eliminar el archivo swap ampliado — el swap causa muchos ciclos de escritura en la microSD
    sudo swapoff /var/swapfile
    sudo rm /var/swapfile
    # Edita /etc/fstab y elimina la línea de swap añadida en el Paso 1

    # Opcionalmente, elimina los directorios fuente para recuperar ~1.5 GB de espacio en disco
    # (conserva si quieres los ejemplos o planeas recompilar)
    # cd ~ && rm -rf opencv opencv_contrib
    ```
</Steps>

<Aside type="note">
**Tiempos de compilación esperados en Jetson Nano:**
- Compilación de 4 núcleos con 8+ GB de memoria (4 GB RAM + 6 GB swap): **~1.5–2 horas**
- Compilación de 1 núcleo (memoria insuficiente): **~4–6 horas**
</Aside>

## Problemas Conocidos

<Aside type="caution">
**Conflicto con la versión de OpenCV preinstalada:** Después de instalar desde fuente, ejecutar `python3 -c "import cv2"` podría seguir importando la versión 4.1.1 de JetPack. Verifica la ruta: `python3 -c "import cv2; print(cv2.__version__, cv2.__file__)"`. Si la ruta muestra `/usr/lib/python3/dist-packages/cv2`, la nueva versión está cargada correctamente. Si muestra otra ruta, ajusta `PYTHONPATH`.

**La cámara CSI con GStreamer no funciona tras una recompilación:** Si recompilas con `-DWITH_GSTREAMER=OFF`, el pipeline de la cámara fallará. Siempre mantén `WITH_GSTREAMER=ON` en el Jetson para soportar la fuente `nvarguscamerasrc` usada con cámaras CSI.

**"No module named cv2.cuda":** El módulo CUDA no fue incluido en la compilación. Vuelve a ejecutar cmake, verifica que `CUDA_ARCH_BIN=5.3` está configurado, que `WITH_CUDA=ON` aparece como `YES` en el resumen de cmake, y que `opencv_contrib` fue clonado con el mismo tag de versión.

**La compilación falla con símbolos CUDA indefinidos:** Generalmente causado por un desajuste de versiones entre los tags de git de `opencv` y `opencv_contrib`. Asegúrate de que ambos repositorios estén en la misma rama: `git -C ~/opencv checkout 4.10.0 && git -C ~/opencv_contrib checkout 4.10.0`

**Advertencia de compatibilidad con NumPy 2.x:** Si encuentras `_ARRAY_API not found` o errores de "compiled against NumPy 1.x", asegura una versión consistente de NumPy: `pip uninstall numpy -y && pip install numpy==2.x`. También añade `-DPYTHON3_NUMPY_INCLUDE_DIRS=$(python3 -c "import numpy; print(numpy.get_include())")` al comando CMake.
</Aside>

## Scripts Alternativos Automatizados

<Aside type="tip">
Existen tres scripts de la comunidad bien mantenidos que pueden usarse como referencia o ejecutarse directamente:

- **Script de Q-engineering:** `github.com/Qengineering/Install-OpenCV-Jetson-Nano` — auto-detecta Nano vs Orin Nano, gestiona la expansión del swap, ejecuta toda la secuencia cmake + build + install. Soporta OpenCV 4.5.3 hasta 4.10.0+.
- **Script de JetsonHacks:** `github.com/JetsonHacksNano/buildOpenCV` — más antiguo pero bien documentado; útil como referencia manual para el conjunto completo de flags.
- **Script oficial de NVIDIA:** `github.com/AastaNV/JEP` (en el directorio `scripts/`) — script de instalación propio de NVIDIA con menos opciones de personalización.
</Aside>

## Comparación: Desktop vs Jetson Nano

| Aspecto | PC Desktop (Linux/Windows) | Jetson Nano |
|---|---|---|
| `CUDA_ARCH_BIN` | Coincide con tu GPU (ej. `8.6` para RTX 3080) | Siempre `5.3` (Maxwell) |
| Versión de CUDA | Cualquier CUDA Toolkit reciente (12.x recomendado) | CUDA 10.2 (JetPack 4.6.x) |
| cuDNN | Descargar por separado, copiar al directorio CUDA | Preinstalado con JetPack |
| Tiempo de compilación | 20–60 minutos (con CPU rápida) | 1.5–4 horas (requiere swap) |
| Gestión de memoria | Raramente es un problema | Crítico — debes ampliar el swap a 6+ GB |
| Ruta de instalación de Python | `/usr/local/lib/python3.x/dist-packages/` | Debes configurar explícitamente `PYTHON3_PACKAGES_PATH` |
| Prefijo de instalación | `/usr/local` (estándar) | `/usr` (coherente con la convención de JetPack) |
| `ENABLE_NEON` | No aplica en x86 | Activar `ON` para aceleración NEON en ARM |
| GStreamer | Opcional | Muy recomendado (cámaras CSI) |
| `BUILD_TESTS` | `ON` para desarrollo | `OFF` para ahorrar tiempo de compilación y disco |
| Eliminar OpenCV antiguo | Opcional | Requerido — el cv2 de JetPack oculta la nueva compilación |
