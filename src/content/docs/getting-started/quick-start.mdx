## Your first OpenCV script

Assuming you already have OpenCV installed, we'll start with a basic example: reading an image from disk, displaying it in a window, and saving it again.

```python
import cv2

# Read an image from disk
img = cv2.imread("image.jpg")  # Replace with your own image path

if img is None:
    raise FileNotFoundError("Could not load image. Check the path.")

# Show the image in a window
cv2.imshow("Original image", img)

# Wait until any key is pressed
cv2.waitKey(0)

# Close all OpenCV windows
cv2.destroyAllWindows()

# Save a copy of the image
cv2.imwrite("image_copy.jpg", img)
```

This example introduces the fundamental functions: `cv2.imread()` to load images, `cv2.imshow()` to display them, and `cv2.imwrite()` to save them.

## Fundamental concepts you should understand

Before diving into more complex operations, it's crucial to understand some fundamental aspects of how OpenCV handles images in Python:

<Steps>
  1. **OpenCV uses BGR, not RGB**

     One of the most important differences is that when you read an image with `cv2.imread()`, the color channel order is **BGR** (Blue, Green, Red), unlike many other libraries like Matplotlib, which use **RGB**.

     ```python
     import cv2

     img_bgr = cv2.imread("image.jpg")
     img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB)
     ```

     <Aside type="caution">
     If you plan to visualize images with Matplotlib after processing them with OpenCV, always convert from BGR to RGB before displaying them.
     </Aside>

  2. **Images are NumPy arrays**

     In Python, OpenCV images are simply multidimensional NumPy arrays (`numpy.ndarray`). This means you can use all NumPy operations directly on images.

     ```python
     import cv2

     img = cv2.imread("image.jpg")

     print(type(img))       # <class 'numpy.ndarray'>
     print(img.shape)       # (height, width, channels)
     print(img.dtype)       # usually uint8
     ```

     <Aside>
     **Important details**:
     - `img.shape` returns a tuple `(height, width, channels)`. For a color image, it will typically be `(height, width, 3)`.
     - By default, 8-bit images use `dtype=uint8`, meaning integer values between 0 and 255.
     </Aside>

  3. **Coordinate system and indexing**

     Pixel indexing in OpenCV follows the `[row, column]` convention, which corresponds to `[y, x]`:

     ```python
     pixel = img[10, 20]  # y = 10, x = 20
     print(pixel)         # [B, G, R] for color images
     ```

     Keep in mind that the first index corresponds to the row (vertical coordinate) and the second to the column (horizontal coordinate).
</Steps>

## Basic image operations

Below, we present common transformations that are part of virtually any computer vision pipeline.

### Converting to grayscale

Converting to grayscale is one of the most frequent operations, as it reduces dimensionality and simplifies many algorithms:

```python
import cv2

img = cv2.imread("image.jpg")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

cv2.imwrite("image_gray.jpg", gray)
```

### Resizing images

OpenCV offers flexibility to resize images in different ways:

```python
import cv2

img = cv2.imread("image.jpg")

# Resize to a fixed size (width, height)
resized = cv2.resize(img, (640, 480))

# Resize with a scale factor (maintains aspect ratio)
smaller = cv2.resize(img, None, fx=0.5, fy=0.5)

cv2.imwrite("image_resized.jpg", resized)
cv2.imwrite("image_smaller.jpg", smaller)
```

### Cropping regions of interest

Cropping allows you to extract specific regions from an image using NumPy indexing:

```python
import cv2

img = cv2.imread("image.jpg")

# Crop region: [y1:y2, x1:x2]
crop = img[100:300, 200:500]

cv2.imwrite("image_crop.jpg", crop)
```

### Drawing shapes and text

OpenCV provides functions to draw on images, useful for visualization and annotation:

```python
import cv2

img = cv2.imread("image.jpg")

# Line: start point, end point, color (B, G, R), thickness
cv2.line(img, (50, 50), (250, 50), (0, 255, 0), 3)

# Rectangle: top-left corner, bottom-right corner, color, thickness
cv2.rectangle(img, (50, 100), (250, 250), (255, 0, 0), 2)

# Circle: center, radius, color, thickness (-1 for solid fill)
cv2.circle(img, (150, 350), 50, (0, 0, 255), -1)

# Text: text, origin, font, scale, color, thickness, line type
cv2.putText(
    img,
    "Hello OpenCV",
    (50, 420),
    cv2.FONT_HERSHEY_SIMPLEX,
    1.0,
    (255, 255, 255),
    2,
    cv2.LINE_AA,
)

cv2.imwrite("image_drawing.jpg", img)
```

## Filters and edge detection

### Gaussian blur

Gaussian blur is useful for reducing noise and smoothing images:

```python
import cv2

img = cv2.imread("image.jpg")

# Kernel size must be odd (e.g., 3, 5, 7, 15, ...)
blur = cv2.GaussianBlur(img, (15, 15), 0)

cv2.imwrite("image_blur.jpg", blur)
```

The third parameter (0) is the standard deviation in X; if set to 0, OpenCV calculates it automatically based on the kernel size.

### Edge detection with Canny

The Canny algorithm is one of the most popular methods for detecting edges in images:

```python
import cv2

img = cv2.imread("image.jpg")
gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# Canny: image, lower threshold, upper threshold
edges = cv2.Canny(gray, 100, 200)

cv2.imwrite("image_edges.jpg", edges)
```

The thresholds control the sensitivity of detection: lower values detect more edges, while higher values are more selective.

## Color spaces

Working with different color spaces is fundamental for tasks like segmentation, object detection, and color analysis:

```python
import cv2

img = cv2.imread("image.jpg")

# Conversions to different color spaces
img_rgb = cv2.cvtColor(img, cv2.COLOR_BGR2RGB)
img_hsv = cv2.cvtColor(img, cv2.COLOR_BGR2HSV)
img_lab = cv2.cvtColor(img, cv2.COLOR_BGR2LAB)

cv2.imwrite("image_rgb.jpg", img_rgb)
cv2.imwrite("image_hsv.jpg", img_hsv)
cv2.imwrite("image_lab.jpg", img_lab)
```

Each color space has its advantages: HSV is useful for color-based segmentation, while LAB is more perceptually uniform.

## Working with video

### Reading a video file

OpenCV allows you to process videos frame by frame:

```python
import cv2

cap = cv2.VideoCapture("video.mp4")  # Replace with your video path

if not cap.isOpened():
    raise RuntimeError("Could not open video file.")

while True:
    ret, frame = cap.read()
    if not ret:
        break

    cv2.imshow("Video", frame)

    # Press 'q' to exit
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

cap.release()
cv2.destroyAllWindows()
```

The `cap.read()` method returns a tuple: a boolean indicating whether the frame was read successfully, and the frame itself.

### Capturing from webcam

To capture real-time video from a webcam:

```python
import cv2

# 0 is usually the default camera
cap = cv2.VideoCapture(0)

if not cap.isOpened():
    raise RuntimeError("Could not open webcam.")

while True:
    ret, frame = cap.read()
    if not ret:
        break

    cv2.imshow("Webcam", frame)

    if cv2.waitKey(1) & 0xFF == ord("q"):
        break

cap.release()
cv2.destroyAllWindows()
```

<Aside type="tip">
Make sure to release the resource with `cap.release()` when finished.
</Aside>

## Complete mini pipeline example

Below, we present an example that integrates several typical operations into a complete pipeline:

1. Reading an image.
2. Resizing.
3. Applying blur.
4. Edge detection.
5. Finding contours.
6. Drawing bounding boxes around detected objects.

```python
import cv2

# 1. Read image
img = cv2.imread("shapes.png")  # Use an image with simple shapes
if img is None:
    raise FileNotFoundError("Could not load shapes.png")

# 2. Resize (optional, to standardize size)
img_resized = cv2.resize(img, (640, 480))

# 3. Convert to grayscale and apply blur
gray = cv2.cvtColor(img_resized, cv2.COLOR_BGR2GRAY)
blur = cv2.GaussianBlur(gray, (5, 5), 0)

# 4. Edge detection
edges = cv2.Canny(blur, 50, 150)

# 5. Find contours
contours, _ = cv2.findContours(edges, cv2.RETR_EXTERNAL, cv2.CHAIN_APPROX_SIMPLE)

# 6. Draw bounding boxes for each contour
output = img_resized.copy()
for cnt in contours:
    x, y, w, h = cv2.boundingRect(cnt)
    cv2.rectangle(output, (x, y), (x + w, y + h), (0, 255, 0), 2)

cv2.imwrite("shapes_detected.jpg", output)
```

This type of pipeline forms the basis of many real-world applications, such as simple object detection, automated visual inspection, and shape analysis.

## Basic face detection with Haar Cascades

Although more modern methods exist (such as those based on deep learning), Haar cascade classifiers remain a simple way to get started with face detection:

```python
import cv2

# Load the pre-trained face detector
face_cascade = cv2.CascadeClassifier(
    cv2.data.haarcascades + "haarcascade_frontalface_default.xml"
)

img = cv2.imread("people.jpg")
if img is None:
    raise FileNotFoundError("Could not load people.jpg")

gray = cv2.cvtColor(img, cv2.COLOR_BGR2GRAY)

# Detect faces: returns a list of tuples (x, y, w, h)
faces = face_cascade.detectMultiScale(gray, scaleFactor=1.1, minNeighbors=4)

# Draw rectangles around detected faces
for (x, y, w, h) in faces:
    cv2.rectangle(img, (x, y), (x + w, y + h), (255, 0, 0), 2)

cv2.imwrite("people_faces.jpg", img)
```

The `scaleFactor` and `minNeighbors` parameters control the sensitivity and accuracy of detection. Adjust these values according to your needs.

## Common problems and best practices

### File paths

If `cv2.imread()` returns `None`, it's almost always due to a file path issue.

**Solution**: Use absolute paths or check the current working directory:

```python
import os
print(os.getcwd())  # Shows the current working directory
```

### Windows that don't appear

In some environments (such as Jupyter Notebooks or certain IDEs), `cv2.imshow()` may not work correctly.

**Solution**: Use Matplotlib to visualize images in these cases:

```python
import cv2
import matplotlib.pyplot as plt

img_bgr = cv2.imread("image.jpg")
img_rgb = cv2.cvtColor(img_bgr, cv2.COLOR_BGR2RGB)

plt.imshow(img_rgb)
plt.axis("off")
plt.show()
```

### Data types and value ranges

Many OpenCV algorithms assume images are in `uint8` format with values between 0 and 255.

<Aside type="caution">
If you normalize images to `float32` or `float64` (values between 0.0 and 1.0), make sure to convert back to the correct range before saving or displaying:
</Aside>

```python
# If working with normalized images
img_normalized = img.astype(np.float32) / 255.0
# ... processing ...
# Convert back to save
img_uint8 = (img_normalized * 255).astype(np.uint8)
cv2.imwrite("output.jpg", img_uint8)
```

### Basic performance

For real-time video processing, optimize your code:

- **Avoid unnecessary operations inside the loop**: Precompute values that don't change between frames.
- **Keep per-frame processing as simple as possible**: Only increase complexity if performance allows.
- **Consider resizing**: Processing smaller frames is significantly faster.

```python
# Example: resize before processing to improve performance
cap = cv2.VideoCapture(0)
while True:
    ret, frame = cap.read()
    if not ret:
        break
    
    # Resize for faster processing
    small_frame = cv2.resize(frame, (320, 240))
    # ... process small_frame ...
    
    # Resize back for display
    display_frame = cv2.resize(small_frame, (640, 480))
    cv2.imshow("Video", display_frame)
    
    if cv2.waitKey(1) & 0xFF == ord("q"):
        break
```
